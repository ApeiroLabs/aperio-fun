# Aperio 内容地图

`aperio.fun` 长期收录三类内容。这个文件是内容方向的单一事实来源，新增栏目或改变发布方式时应同步更新。

原始内容先分别放入：

- `content-inbox/daily-news/`
- `content-inbox/inspiration/`
- `content-inbox/essays/`

这些目录是投放区，不要求文件预先符合网站格式。

## 1. 每日新闻

- 来源：由 Horizon 产生。
- 原始参考会话：`019efc71-52be-7041-bf1f-7626eca3345e`
- 定位：经过筛选和整理的每日信息，而不是未经编辑的链接堆积。
- 发布方式：待参考会话可访问后，根据其实际输出格式设计自动导入和发布流程。
- 建议标识：`daily-news`

## 2. 灵感

- 来源：灵感记录与后续整理。
- 原始参考会话：`019f02c0-16ed-7bc2-b3f7-d6aea1687f68`
- 定位：短想法、观察、摘记以及尚未发展成完整文章的种子。
- 发布方式：待参考会话可访问后，确认字段、长度和整理流程。
- 建议标识：`inspiration`

## 3. 自己写的文章

- 来源：本人写作。
- 定位：经过完整组织和编辑的长文、随笔与实践记录。
- 发布方式：直接在 `src/content/blog/` 中新增 Markdown 文件。
- 建议标识：`essay`

## 信息架构原则

- 首页可以混合展示三类内容，但必须清楚显示内容类型。
- `/writing/` 是所有内容的总入口，并提供三类内容的独立入口。
- 每日新闻不应在视觉上冒充本人原创文章。
- 灵感允许短小和未完结，但应保留发布日期和来源上下文。
- 本人文章拥有最高的编辑权重和最安静的阅读版式。
- 自动生成内容发布前必须经过明确的质量检查，不把流水线原始输出直接暴露给读者。

## 待确认

- Horizon 当前以同日期 `zh/en` 两份 Markdown 发布，网站按日期归并并提供语言切换。
- 灵感当前是中文 Markdown 长报告。
- 三类内容使用独立路径：`/news/`、`/inspiration/` 和 `/writing/`。

## 全局语言规则

- 中文是默认语言，使用 `/`、`/writing/`、`/news/` 和 `/inspiration/`。
- 英文统一使用 `/en/` 前缀。
- 页面只展示当前语言的内容，不跨语言回退。
- Horizon 通过文件名末尾的 `-zh` 或 `-en` 标记语言。
- 灵感默认中文，英文文件名必须以 `-en.md` 结尾。
- 自己写的文章在 frontmatter 中使用 `lang: zh` 或 `lang: en`。
- 语言切换优先前往对应译文；没有译文时前往另一语言的同栏目列表。
