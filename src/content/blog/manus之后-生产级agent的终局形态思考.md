4 月 27 日，发改委叫停了 Meta 对 Manus AI 的 20 亿美金收购，但我认为就算收购没被叫停，Manus 这条路也走到了尽头。

收购搁浅只是提前宣判了死刑。真正推向它走向末路的是其赖以成名的“一力降十会”。

## 一、从 Monica 到 Manus：美丽的泡沫

“如果说 ChatGPT 的发布是 AI 应用发展的‘语言起源’（赋予了 AI 表达思想的能力），那 Manus 则可以算是‘工具革命’（赋予了 AI 改造现实的手脚）。”

作为最早使用 Monica 的用户，到后续的 Manus 的爆火（至少在中文区），我认为 Manus 在 2025 年讲了一个非常成功的故事。它没有卷高投入的模型参数，而是从产品范式上完成了降维打击。

### 视觉层（想象力）：把 Agent 从黑盒做成了可视化的“数字员工”

在它之前，AutoGPT 那一代 Agent 的执行过程只存在于控制台日志里。Manus 把这套逻辑塞进了云端虚拟桌面，或者直接操作你的浏览器：鼠标在哪、点了什么、卡在哪个弹窗。这种视觉闭环第一次让普通用户相信 AI 真的“在干活”，而不是在编故事。

### 执行层（技术闭环）：吃下了 90% 没有 API 的长尾互联网

因为互联网 90% 的场景没有标准接口，所以 Manus 押注的是：只要能搞定一个通用的浏览器操作协议，就能应对所有的业务场景。作为一个做浏览器起家的团队，过去的技术积累形成了短期内的护城河。

即使以今天的视角来看，Manus 仍然有相当多的受众：标准 Agent 接入第三方系统要走 API、要审批、要 OAuth。Manus 借用用户的浏览器登录态进行操作，无需任何 IT 配合就能介入工作流的方式，在用户体验上简直是降维打击。

这个路线让 Manus 在 2025 年同时拿到了“产品想象力”和“现金流”，避开模型竞赛的同时赚足了眼球，几乎拿到了“结果”。

## 二、暴力美学的不可持续：内忧外患

故事美好，账面流水漂亮（100M ARR），但是团队要卖掉，到底是哪里出了问题呢，总不能是 20 亿亏本卖给你交个朋友吧。

### Token 经济学与视觉税

让 AI 看见、操作网页是要烧钱的。每点一次鼠标，都要把复杂的网页翻译给 AI 听，再加上随着操作步骤指数增长的上下文膨胀压力，都会带来严重财务压力。

在 Demo 里跑通任务是惊喜，但抛开了尝鲜阶段和补贴红利，生产环境的执行成本覆盖了业务得利，就很难称得上是好的生意了。

### UI 是全世界最通用，但也是最烂的 API

网页 UI 是给人眼设计的，充满了模糊与不确定性（前端逻辑五花八门，前端开发者水平也是参差不齐）。API 调用有明确的、结构化的结果，但浏览器操作没有。

视觉反馈与真实状态的脱钩，让 AI 在处理弹窗、验证码或复杂表单时，极易陷入死循环。更致命的是，这种“猜测式”的逻辑在特殊的场景等于灾难：金融市场上 1% 的随机失败率，等于 100% 的业务事故。

### 上有大厂，下有开源，时代变了大人

如果说技术债是内忧，2026 年的行业变化更让人头疼。

**旧时王谢堂前燕，飞入寻常百姓家。** 开源把 Agent 平民化了。OpenClaw 在 GitHub 几周斩获数万 star，虽然同样是昙花一现，但也给所有人埋下了一个种子：开源框架封装加模型封装，任何人都能轻松拥有一个私人 AI 助手。当技术泡沫被打破，Manus 昂贵的订阅瞬间失去了性价比。

**协议霸权。** 通过 MCP，大厂正在给全世界的工具装上“统一插头”。当 Claude 可以直接通过协议读取你的数据库索引、调用 IDE 的底层 API 时，它不需要像 Manus 那样在云端虚拟机里像个盲人摸象一样解析前端组件。当 Manus 还在研究如何配一把好用的万能钥匙时，房东直接把门打开了。

Manus 陷入了最难的位置，技术护城河被打破后，小扎发起了仅退款（世上真的有后悔药啊）。

## 三、生产级的 Agent 长什么样？

即使 Manus 有诸多的问题，但是我们也要承认，Manus 团队有着行业顶尖的产品嗅觉。当大家还在纠结怎么写 Prompt 才能让机器人说话更好听时，Manus 已经让用户体验到了“看别人干活”的快感。

未来的超级应用不应该是一个对话框，而应该是一个“结果交付引擎”。这种从“对话框”到“执行器”的跨越，是 2026 年所有 Agent 产品的认知基石。

作为行业从业者，我也常常思考一个能撑起商业闭环的 Agent 该是什么样子，用户需求是什么，呈现效果又是什么。从市场已有的产品来看：

**Manus 类，即插即用的“拎包入住”感：** 登录、充值、开始。它把复杂的云端环境封装成了一个极其精美的网页。这种“消费级”的门槛，是它能迅速破圈的核心原因。它卖的不只是技术，更是省心：“实时直播”带来的信任感和快感是无与伦比的，结果呈现方式也更自由。

**Claude Code / Cursor 类，本地化的影子学徒：** 形态是桌面 CLI 或 IDE 插件，逻辑是“结对共生”。它就在你手边看着你的文件，反应极快，像是一个坐在你身后实时待命的实习生。但这种体感极其依赖“宿主”：它面临严重的权限焦虑与离线缺失。你下班它也得断电，你断网它就失灵。它本质上不是一个能独立交付结果的执行实体，而是大模型能力在本地环境的一种高频延伸。

**各种 Claw 托管，租用的透明工厂：** 形态通常是 Telegram 或 Discord 加上一个简陋的 Dashboard，逻辑是“消息触发制”。它的执行过程全透明、可审计，就像一个能随时进出的加工车间。但致命伤在于交互的破碎感：IM 界面根本承载不了复杂的资产交付。当 Agent 正在监控金融异动并生成风险报告时，几十条碎片化的消息推送本身就是一场交互灾难。它虽然有硬核的生产力，却完全没有“产品感”。

以上分类并非死板的功能割裂。事实上，Manus 已经增加了其 IM 触达能力，Cursor 也推出了云端托管方案。这里的分类更多是代表各家产品的初始路径。讨论的是不同范式背后的核心逻辑，而非单一的功能有无。

我认为能撑起生产场景的 SaaS Agent，产品形态和能力要包括：

**消灭配置，“零负担”准入，登录即用。** 拒绝 Docker 镜像，拒绝 Python / TS 环境，拒绝 API Key，用户面对的永远是一个美观的网页。所有的环境搭建、依赖安装、云端虚拟机分配，全部在后台静默完成。用户的时间应该花在下指令上，而不是花在修补环境上。

**云端“心脏”，7×24 的断网自由。** 拥有独立的执行空间，后台执行是 Agent 的基本尊严。合上电脑去睡觉，它在后台独自完成长达数小时的任务。你不需要担心网页是否会崩溃，也不需要陪着它“监工”。

**多端“神经”，IM 实时决策与通知。** Agent 不应该只是一个死板的标签页，它应该长在你的社交、办公流里。它是跨平台的。IM 界面不是用来交互复杂的逻辑，而是用来做“高频决策”和“实时进度通报”。它在关键时刻“拍拍你”，让你在不增加摩擦的前提下，始终拥有对任务的最终解释权。

**网页“工作站”，从“对话框”到“作战指挥室”。** 即使是各种 Claw 发展的今天，我也始终认为网页端是生产应用的必选之路，仅靠 IM 的碎片化通知不足以承载确定性的任务编排、进度追踪和结果展示。

## 四、终局：从表演到工业闭环

Manus 的爆火，是因为它让普通人看到了 Agent 有多“美”：极致的 SaaS 化体验和视觉反馈，让 AI 摆脱了冰冷的对话框；Claude Code、OpenClaw、Hermes 等流行，又让人们看到了 Agent 的“脑”有多快：基于原生协议的执行效率，证明了 Agent 不需要模仿人也能改变世界。

商业闭环并不会是任何单一形态的极端，而是那个能把 Manus 的“拎包入住”、Claude Code 的“Agent 原生”，以及 Claw SaaS 的“云端常驻”完美缝合在一起的产品。这也标志着 Agent 正式从“魔术演示”转向“确定性的工业闭环”。它不再试图在视觉上扮演一个“替身”，而是作为一个高效的数字器官，长在云端，连在指尖，彻底解放生产力。
